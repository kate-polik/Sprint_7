
# Тестирование API учебного сервиса Яндекс Самокат

Проект предназначен для проверки API учебного сервиса Яндекс Самокат.  
Документация API доступна по адресу: [qa-scooter.praktikum-services.ru/docs/](https://qa-scooter.praktikum-services.ru/docs/)

## Что проверяется

### 1. Создание курьера (файл `test_creating_courier.py`)
Тесты для проверки создания курьера.

#### Проверки:
- Курьера можно создать;
- Чтобы создать курьера, нужно передать в ручку все обязательные поля;
- Запрос возвращает правильный код ответа;
- Успешный запрос возвращает `{"ok":true}`.

**Реализовано в тесте:** `test_create_courier_success`.

#### Проверки:
- Нельзя создать двух одинаковых курьеров;
- Если создать пользователя с логином, который уже есть, возвращается ошибка.

**Реализовано в тесте:** `test_create_duplicate_couriers`.

#### Проверки:
- Если одного из полей нет, запрос возвращает ошибку.

**Реализовано в тесте:** `test_create_courier_missing_fields`.

---

### 2. Логин курьера (файл `test_courier_login.py`)
Тесты для проверки авторизации курьера.

#### Проверки:
- Курьер может авторизоваться;
- Для авторизации нужно передать все обязательные поля;
- Успешный запрос возвращает `id`.

**Реализовано в тесте:** `test_courier_login_success`.

#### Проверки:
- Система вернёт ошибку, если неправильно указать логин или пароль;
- Если какого-то поля нет, запрос возвращает ошибку;
- Если авторизоваться под несуществующим пользователем, запрос возвращает ошибку.

**Реализовано в тесте:** `test_login_negative_cases`.

---

### 3. Создание заказа (файл `test_orders.py`)
Тесты для проверки создания заказа.

#### Проверки:
- Можно указать один из цветов — `BLACK` или `GREY`;
- Можно указать оба цвета;
- Можно совсем не указывать цвет;
- Тело ответа содержит `track`.

**Реализовано с использованием параметризации в тесте:** `test_create_order`.

---

### 4. Список заказов (файл `test_orders.py`)
Тесты для проверки получения списка заказов.

#### Проверки:
- Тело ответа возвращает список заказов.

**Реализовано в тесте:** `test_get_orders`.

---

## Структура проекта

Проект имеет следующую структуру:
```
Sprint_7/
├── allure_results/              # Папка для хранения результатов тестов Allure
├── tests/
│   ├── test_courier_login.py    # Тесты логина курьера
│   ├── test_creating_courier.py # Тесты создания курьера
│   ├── test_orders.py           # Тесты обработки заказав
├── utils/
│   ├── helpers.py               # Вспомогательные функции
├── config.py                    # Конфигурация с URL
├── conftest.py                  # Фикстуры для тестов
├── README.md                    # Описание проекта                   
├── requirements.txt             # Список зависимостей для проекта
```

## Особенности реализации

- **Корректный нейминг:** Все элементы проекта названы в соответствии с правилами.
- **Разделение тестов:** Для каждой ручки API тесты находятся в отдельном классе.
- **Покрытие тестами:**
  - Написаны тесты на ручку «Создать курьера».
  - Написаны тесты на ручку «Логин курьера».
  - Написаны тесты на ручку «Создать заказ».
  - Написаны тесты на ручку для получения списка заказов.
- **Allure-отчёт:** Сгенерирован отчёт с использованием Allure.
- **Проверки кода и тела ответа:** Во всех тестах проверяется как код ответа, так и его тело.
- **Независимость тестов:** Все тесты независимы друг от друга.
- **Обработка тестовых данных:**
  - Необходимые данные создаются перед тестом.
  - Данные удаляются после завершения теста.
